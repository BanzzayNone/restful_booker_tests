# Restful Booker API Testing Framework

Автоматизированное тестирование API отеля [restful-booker.herokuapp.com](https://restful-booker.herokuapp.com/apidoc/).

Этот учебный проект демонстрирует мой взгляд и возможности API-тестирования: параметризация, Contract Testing, негативные сценарии и независимость тестов. Ввиду ограничений API пополняться не будет, Playwright и прочие пользуемые мной библиотеки будут представлены в других проектах.

---

## Что внутри

| Функция              | Описание                                                     |
| -------------------- | ------------------------------------------------------------ |
| **CRUD-тесты**       | Полное покрытие: Create, Read, Update, Delete                |
| **Параметризация**   | 1 тест = 10+ прогонов с разными данными (Faker)              |
| **Contract Testing** | Валидация структуры ответов через Pydantic                   |
| **Негативные тесты** | Валидация ошибок, граничные значения, авторизация            |
| **Независимость**    | Тесты можно запускать в любом порядке и по маркерам          |
| **Генерация данных** | Реалистичные данные через Faker (имена, даты, числа)         |
| **Фикстуры**         | Переиспользуемые компоненты: клиент, токен, данные           |
| **Cleanup**          | Автоматическая очистка данных через `create_booking` fixture |

## Установка и запуск

### 1. Виртуальное окружение

```bash
# Создать виртуальное окружение
python -m venv .venv

# Активировать окружение
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

```bash
# Скопировать шаблон
cp .env.example .env

# Отредактировать .env при необходимости (дефолтные значения — публичные credentials демо-API)
```

### 3. Запуск тестов

```bash
# Все тесты
pytest

# Только smoke-тесты (быстрая проверка)
pytest -m smoke -v

# Только негативные тесты
pytest -m negative -v

# Только Contract-тесты
pytest -m contract -v

# Только regression-тесты
pytest -m regression -v

# С отчётом в HTML
pytest --html=reports/report.html -v

# Параллельный запуск
pytest -n auto
```

### 4. Фильтрация по файлам

```bash
# Конкретный файл
pytest tests/API/test_negative.py -v

# Конкретный тест
pytest -k "test_name_example"

# Конкретный параметр (из параметризации)
pytest "tests/API/test_negative.py::test_create_booking_with_invalid_price[invalid_price0]" -v
```

## Основы тест-дизайна в проекте

### 1. Минимум тестов, максимум покрытия

Один параметризованный тест заменяет 10+ отдельных функций. Данные генерируются через Faker для реалистичности и разнообразия.

### 2. Независимость тестов

- Каждый тест создаёт и удаляет свои данные (через `create_booking` fixture с автоматическим cleanup)
- Тесты можно запускать в любом порядке
- Поддержка параллельного запуска (`pytest -n auto`)

### 3. Contract Testing

Валидация структуры ответов через Pydantic-модели для удобства чтения и модификации.

### 4. Централизованная конфигурация

Все настройки в одном месте (`config/settings.py` + `.env`) для быстрого изменения. Единая фабрика тестовых данных `make_booking_payload()`.

## Ограничения API

При написании негативных сценариев опытным путём выявлены недостатки учебного API:

1. **Отсутствие валидации цены** — API принимает любые значения `totalprice` (отрицательные, строки). Тесты помечены `xfail`.
2. **Некорректная обработка пустых полей** — сервер возвращает 500 вместо 400 при отсутствии обязательных полей.
3. **Проблема с авторизацией** — при невалидном токене API возвращает 403 вместо 401.
4. **Отсутствие валидации Content-Type** — API не блокирует запросы с неверным форматом заголовка.
